◆ nvm_logger_log_init()

uint32_t nvm_logger_log_init ( log_config_t *  log_config)

Function to setup the log.

Parameters
log_idUnique Log ID which is to be used to keep track of specific data type. @Note log_id has to be less than NVM_LOGGER_MAX_LOGS. If log_id is not\ specified then module will assign the log_id automatically.
entry_sizeSize of data_type which is to be stored. Default value is 32
log_sizeMaximum number of data entries that user wants to keep. Default value is 254 @Note If log_size is not specified but data_size is specified other\ than 32 then logger store maximum number of logs possible in one page only i.e\ 4080/data_size logs
Returns
log_id Log ID which is supposed to used for the log which is being setup.
Return values
updated_log_idIf log_id is not specified or having conflicts with previously\ declared logs, module will update the log_id.
original_log_idIf log_id is available then it'll not get updated.
