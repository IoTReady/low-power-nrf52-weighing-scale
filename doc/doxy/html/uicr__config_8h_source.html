51 /*
52  * Include this file in your project if you want to include in your compiled code files data
53  * for the User Information Configuration Registers (UICR) area; see nRF51 Series Reference
54  * Manual chapter User Information Configuration Registers. This file declares one variable
55  * per register of the UICR area and informs the linker where to place them. To include
56  * the desired value in the desired address, uncomment the variable with the proper address
57  * at the target area and update the assignment value.
58  *
59  * Please note that UICR values are stored in a reserved area of the flash and should only be
60  * stored into when downloading a hex file. Do not use these defined variables to store data
61  * at run time.
62  *
63  * Note as well that this file uses one non-standard attribute ("at"). It will only function
64  * with the ARMCC compiler toolset.
65  *
66  * Note that the hex file generated when this file is included will fail to download when using
67  * the standard download algorithm provided by Nordic. See example project "uicr_config_example"
68  * in any of the board example folders for an example of the recommended download method as well
69  * as the documentation that follows with the SDK. nrfjprog can be used as normal.
70  *
71  * Please note as well that if you are using a SoftDevice the UICR_CLENR0 address will
72  * already be in use. Do not uncomment that line.
73  */
