◆ nrf_drv_twi_xfer()

__STATIC_INLINE ret_code_t nrf_drv_twi_xfer ( nrf_drv_twi_t const *  p_instance,
nrf_drv_twi_xfer_desc_t const *  p_xfer_desc,
uint32_t  flags 
)

Function for preparing a TWI transfer.

The following transfer types can be configured (nrf_drv_twi_xfer_desc_t::type):

  • NRF_DRV_TWI_XFER_TXRX<span>: Write operation followed by a read operation (without STOP condition in between).
  • NRF_DRV_TWI_XFER_TXTX<span>: Write operation followed by a write operation (without STOP condition in between).
  • NRF_DRV_TWI_XFER_TX<span>: Write operation (with or without STOP condition).
  • NRF_DRV_TWI_XFER_RX<span>: Read operation (with STOP condition).

Additional options are provided using the flags parameter:

  • NRF_DRV_TWI_FLAG_TX_POSTINC and NRF_DRV_TWI_FLAG_RX_POSTINC<span>: Post-incrementation of buffer addresses. Supported only by TWIM.
  • NRF_DRV_TWI_FLAG_NO_XFER_EVT_HANDLER<span>: No user event handler after transfer completion. In most cases, this also means no interrupt at the end of the transfer.
  • NRF_DRV_TWI_FLAG_HOLD_XFER<span>: Driver is not starting the transfer. Use this flag if the transfer is triggered externally by PPI. Supported only by TWIM. Use nrf_drv_twi_start_task_get to get the address of the start task.
  • NRF_DRV_TWI_FLAG_REPEATED_XFER<span>: Prepare for repeated transfers. You can set up a number of transfers that will be triggered externally (for example by PPI). An example is a TXRX transfer with the options NRF_DRV_TWI_FLAG_RX_POSTINC, NRF_DRV_TWI_FLAG_NO_XFER_EVT_HANDLER, and NRF_DRV_TWI_FLAG_REPEATED_XFER. After the transfer is set up, a set of transfers can be triggered by PPI that will read, for example, the same register of an external component and put it into a RAM buffer without any interrupts. nrf_drv_twi_stopped_event_get can be used to get the address of the STOPPED event, which can be used to count the number of transfers. If NRF_DRV_TWI_FLAG_REPEATED_XFER is used, the driver does not set the driver instance into busy state, so you must ensure that the next transfers are set up when TWIM is not active. Supported only by TWIM.
  • NRF_DRV_TWI_FLAG_TX_NO_STOP<span>: No stop condition after TX transfer.
Note
Some flag combinations are invalid:
  • NRF_DRV_TWI_FLAG_TX_NO_STOP with nrf_drv_twi_xfer_desc_t::type different than NRF_DRV_TWI_XFER_TX
  • NRF_DRV_TWI_FLAG_REPEATED_XFER with nrf_drv_twi_xfer_desc_t::type set to NRF_DRV_TWI_XFER_TXTX

If nrf_drv_twi_xfer_desc_t::type is set to NRF_DRV_TWI_XFER_TX and the NRF_DRV_TWI_FLAG_TX_NO_STOP and NRF_DRV_TWI_FLAG_REPEATED_XFER flags are set, two tasks must be used to trigger a transfer: TASKS_RESUME followed by TASKS_STARTTX. If no stop condition is generated, TWIM is in SUSPENDED state. Therefore, it must be resumed before the transfer can be started.

Note
This function should be used only if the instance is configured to work in non-blocking mode. If the function is used in blocking mode, the driver asserts.
If you are using this function with TWI, the only supported flag is NRF_DRV_TWI_FLAG_TX_NO_STOP. All other flags require TWIM.
Parameters
[in]p_instancePointer to the driver instance structure.
[in]p_xfer_descPointer to the transfer descriptor.
[in]flagsTransfer options (0 for default settings).
Return values
NRF_SUCCESSIf the procedure was successful.
NRF_ERROR_BUSYIf the driver is not ready for a new transfer.
NRF_ERROR_NOT_SUPPORTEDIf the provided parameters are not supported.
NRF_ERROR_INTERNALIf an error was detected by hardware.
NRF_ERROR_INVALID_ADDRIf the EasyDMA is used and memory adress in not in RAM
NRF_ERROR_DRV_TWI_ERR_OVERRUNIf the unread data was replaced by new data (TXRX and RX)
NRF_ERROR_DRV_TWI_ERR_ANACKIf NACK received after sending the address.
NRF_ERROR_DRV_TWI_ERR_DNACKIf NACK received after sending a data byte.
